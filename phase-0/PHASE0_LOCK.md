# AGW Phase 0 Lock

Date: 2026-04-14
Project: A.G. Williams Painting
Purpose: Reset scope, lock direction, and define the first controlled client checkpoint before build work.

## 1. Current-site audit summary

### What the live site is really selling
- Heritage first: `since 1906`, family-owned, fourth-generation legacy.
- Trust first: insured, background-checked crews, warranties, premium materials, clear communication.
- Estimate flow first: call or form, estimator visit, fair quote, job done right.
- Geographic proof first: Westchester, Fairfield, Rockland, Putnam.
- Service breadth second: residential, commercial, wallpaper, cabinets, coatings.

### What is working
- Strong trust stack already exists.
- Clear 3-step estimate narrative is easy to preserve.
- Service-area breadth supports both conversion and SEO.
- History/legacy gives AGW a real edge over generic painting sites.

### What is dragging the site down
- Homepage feels repetitive and broad instead of premium and controlled.
- Sitemap is bloated with specials, thin location pages, thank-you pages, archive clutter, and old promo junk.
- Booking UX is operationally safe but visually weak.
- Content hierarchy is muddy: too many paths, too much equal-weight navigation, not enough signal.

### Confirmed integrations already in play
- Google Tag Manager: `GTM-W559QJ7C`
- LeadConnector / GoHighLevel widgets
- Facebook customer chat
- Hotjar
- Rank Math sitemap structure on the current WordPress site

## 2. Approved visual direction board

Direction name: Heritage Precision

### Visual intent
- Quietly premium, bright, restrained, established.
- More luxury service operator than generic contractor.
- Trust and clarity beat hype and busy layouts.
- New England / Fairfield / Westchester polish, not startup SaaS and not blue-collar stock-photo spam.

### Approved styling rules
- Lead with brand heritage, not discount language.
- Use pattern texture lightly in dividers, cards, and background moments only.
- Use bright editorial imagery or real project work, never fake “crew smiling at camera” filler.
- Keep typography clean and calm.
- Let trust elements feel expensive and intentional.

## 3. Approved vs risky assets

### Approved for direction work now
- Logo source files in `brand-kit/01-logo/`
- Pattern files in `brand-kit/03-patterns/`
- Brand reference files in `brand-kit/05-brand-reference/`
- Mock/reference assets in `brand-kit/06-possible-site-assets/` for internal exploration only

### Risky or non-approved
- Everything in `brand-kit/07-risky-or-review-needed/`

### Why those assets are risky
- Ladder imagery creates avoidable safety/insurance optics.
- Ceiling-painting shots feel operationally risky and visually cheap.
- Generic smiling-stock-people imagery weakens trust instead of improving it.
- None of those images support the premium-heritage positioning.

### Rule
- No risky asset enters client-facing comps or site build without explicit AGW approval.

## 7. CMS/editor plan

### Recommended CMS
- Sanity

### Why
- Easier for a marketing team than a bespoke admin.
- Cleaner structured models for services, towns, testimonials, blog, and projects.
- Safer editorial control than trying to recreate WordPress sprawl.
- Good preview workflow for review checkpoints.

### Content models
- Site settings
- Homepage
- Service page
- Commercial page
- Town / service area page
- Testimonial
- Project / gallery item
- Blog post
- FAQ set
- CTA / promo banner
- Team member

## 10. What the first client send should ask

- Does this visual direction feel like the right level of premium and trust for AGW?
- Does this homepage direction reflect the kind of company you want prospects to feel they are hiring?
- Does this estimate/contact direction match how you want new inquiries to come in?

## 11. What the first client send should not ask

- Open-ended sitemap debates
- Broad style brainstorming
- CMS preference fishing
- Copy nitpicks on unfinished pages
